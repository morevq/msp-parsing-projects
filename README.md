# Совместный проект: Сборник парсеров

Добро пожаловать в общий репозиторий проекта! Здесь мы собираем коллекцию скриптов для парсинга данных на Python и практикуем совместную работу с помощью Git.

## Инструкция: что нужно сделать каждому участнику

Вам нужно выполнить следующие шаги, чтобы добавить себя и свой скрипт в общий проект.

### Шаг 1: Создайте свою рабочую ветку
Откройте терминал и создайте новую ветку. Назовите ее своим именем (например, `ivan-branch`):
```bash
git checkout -b <имя-вашей-ветки>
```

### Шаг 2: Добавьте себя в список участников
Откройте файл `main.py`. Найдите список `students` и добавьте туда свое имя и первую букву фамилии (например, `"Алексей М."`). Сохраните файл.

### Шаг 3: Добавьте свой скрипт парсера
В этой же папке создайте новый Python-файл (например, `parser_aleksey.py` или `avito_parser.py`). 
Поместите в этот файл ваш код для парсинга данных.

### Шаг 4: Сделайте коммит
Сохраните изменения в истории Git:
```bash
git add main.py
git add <имя_вашего_файла_парсера>.py
git commit -m "Добавлено имя Алексей и скрипт парсера"
```

### Шаг 5: Влейте свои изменения в `main` (Merge)
Теперь нужно перенести вашу работу в главную ветку:
1. Переключитесь обратно на главную ветку:
   ```bash
   git checkout main
   ```
2. Влейте изменения из вашей рабочей ветки в `main`:
   ```bash
   git merge <имя-вашей-ветки>
   ```

### Шаг 6: Разрешение конфликтов (Самое интересное!)
Так как вы все будете редактировать один и тот же файл `main.py` (добавляя себя в конец одного и того же списка), после того как первый человек сольет свою ветку, у остальных **обязательно возникнет конфликт слияния** (merge conflict)! 
Это нормальная и очень важная ситуация в командной разработке.

Что делать при конфликте:
1. Git сообщит о конфликте в файле `main.py`. Откройте этот файл.
2. Вы увидите специальные метки (<<<<<<<, =======, >>>>>>>), которые показывают вашу версию и версию из `main`.
3. Отредактируйте код вручную так, чтобы в списке оказались имена **всех** участников (и ваше, и тех, кто слил код до вас). Удалите все странные метки Git.
4. Сохраните файл `main.py`.
5. Сделайте коммит, чтобы зафиксировать решение конфликта:
   ```bash
   git add main.py
   git commit -m "Разрешен конфликт: объединены имена участников"
   ```

Готово! Вы успешно справились с добавлением своего кода в общий проект и научились решать конфликты — один из самых важных навыков!
